## 1. 相似度去重

**文件**: `src/dedup/` | **策略**: 两阶段混合去重

### 1.1 第一阶段：dHash（差分哈希）

dHash（Difference Hash）是一种快速的图片感知哈希算法，对图片缩放、亮度变化有一定的鲁棒性。

#### 算法步骤

```
输入: 原始图片 (W × H)
  │
  ├─ 1. 转为灰度图
  ├─ 2. 缩放到 9×8 像素
  ├─ 3. 计算差分: 每行相邻像素比较
  │     如果 pixel[x] > pixel[x+1] → 1
  │     如果 pixel[x] < pixel[x+1] → 0
  └─ 4. 得到 9×8 = 72 对比较 → 72 位 → 截取前 64 位
      输出: 64 位二进制字符串（如 "10100101..."）
```

#### 海明距离

两图片的 dHash 之间的**海明距离** = 对应位不同的数量。

- **距离 = 0**: 两张图片完全一致
- **距离 < 5**: 高度相似（大概率是同一场景的连拍）
- **距离 < 10**: 可能相似
- **距离 ≥ 10**: 大概率不同

#### 阈值说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `dhash_threshold` | 5 | 海明距离 < 此值视为相似，只保留第一张 |

#### 优点与局限

| 优点 | 局限 |
|------|------|
| 极快（~10ms/张 CPU） | 对裁剪/旋转不鲁棒 |
| 不需要 GPU | 对构图相似但内容不同的照片可能误判 |
| 内存占用极低 | |

### 1.2 第二阶段：CLIP 向量相似度

CLIP（Contrastive Language-Image Pre-training）是 OpenAI 的多模态模型，将图片映射到 512 维语义向量空间。

#### 算法步骤

```
输入: 去重后的图片列表
  │
  ├─ 1. 加载 CLIP ViT-B/32 模型（首次需下载 ~600MB）
  ├─ 2. 每张图片预处理: Resize(224) → CenterCrop(224) → Normalize
  ├─ 3. 通过 ViT 编码器得到 512 维嵌入向量
  └─ 4. 计算余弦相似度矩阵
       │
       ├─ 对每对图片: cos_sim = (A·B) / (|A|×|B|)
       └─ 如果 cos_sim > threshold → 标记为相似，删除
```

#### 余弦相似度

```
cos_sim(A, B) = Σ(Ai × Bi) / √(ΣAi²) × √(ΣBi²)

范围: [-1, 1]
  - 1.0: 完全一致
  - > 0.95: 高度语义相似
  - > 0.92: 相似（默认阈值）
  - < 0.8: 大概率不同
```

#### 与 dHash 的对比

| 维度 | dHash | CLIP |
|------|-------|------|
| 速度 | ~10ms/张 | ~100ms/张（CPU） |
| 精度 | 低（像素级） | 高（语义级） |
| 模型大小 | 无 | ~600MB |
| GPU 需求 | 无 | 可选 |
| 对裁剪鲁棒 | ❌ | ✅ |
| 对旋转鲁棒 | ❌ | ✅ |
| 对亮度鲁棒 | ✅ | ✅ |

#### 混合策略优势

```
第一阶段 dHash 粗筛:
  3000张 → 快速过滤高度相似连拍 → ~800张
  (耗时 ~30 秒)

第二阶段 CLIP 精筛:
  ~800张 → 语义去重（不同构图但同一场景） → ~500张
  (耗时 ~80 秒)
```

- **为什么先 dHash 再 CLIP？** CLIP 处理 3000 张需要 5 分钟，先 dHash 降到 800 张后只需 80 秒
- **为什么不只用 CLIP？** CLIP 对连拍（几乎一样的照片）的区分能力不如 dHash

## 2. 美学评分

**文件**: `src/aesthetics/` | **策略**: NIMA 模型为主 + 启发式规则降级

### 2.1 NIMA（Neural Image Assessment）

NIMA 是一个基于 MobileNetV2 的深度学习模型，预测图片的美学评分分布（1-10 分）。

#### 模型架构

```
输入: 224×224 RGB 图片
  │
  ├─ MobileNetV2 backbone（在 ImageNet 上预训练）
  │   1280 维特征向量
  ├─ Dropout(0.75)
  ├─ Linear(1280 → 10)  → 输出 10 个类别的概率分布
  └─ Softmax
       │
       输出: [p1, p2, ..., p10]（Σp = 1）
       
       最终评分 = Σ(pi × i)  (i = 1..10)
```

#### 评分含义

| 分数范围 | 含义 | 示例 |
|----------|------|------|
| 8-10 | 专业级 | 构图精美、光线完美 |
| 6-8 | 优秀 | 旅行照片中的佳作 |
| 4-6 | 普通 | 大多数日常照片 |
| 2-4 | 较差 | 模糊、构图差 |
| 1-2 | 极差 | 几乎全黑/全白 |

#### 降级策略

当 NIMA 模型加载失败（如 torch 未安装、模型下载失败）时，自动降级到启发式规则。

### 2.2 启发式规则评分

无需深度学习，仅基于 OpenCV 图像分析。

#### 四个维度

**① 亮度评分（权重 25%）**

```
将图片转为灰度 → 计算直方图（256 bins）
  ├─ 过曝比例: 亮度 > 230 的像素占比
  └─ 欠曝比例: 亮度 < 25 的像素占比
  
score = 1.0 - (over_exposed + under_exposed)
```

- 理想照片的直方图接近正态分布
- 过曝/欠曝照片评分低

**② 清晰度评分（权重 30%）**

```
将图片转为灰度 → 拉普拉斯算子（二阶导数）
  → 计算方差
  
score = 1 / (1 + exp(-0.05 × (variance - 50)))
```

- `variance > 100`: 清晰
- `variance < 20`: 模糊
- 使用 sigmoid 函数将方差映射到 0-1

**③ 色彩饱和度评分（权重 25%）**

```
转为 HSV 色彩空间 → 取 S 通道均值
  
if s < 0.1:  score = 0.1       # 几乎无色
if s < 0.3:  score = s × 2     # 线性提升
if s < 0.7:  score = 1.0       # 理想范围
else:        score = 1 - (s - 0.7) × 1.5  # 过高则下降
```

- 饱和度在 0.3-0.7 之间最好
- 过低的照片显得平淡，过高的照片显得假

**④ 构图评分（权重 20%）**

```
Sobel 边缘检测 → 计算边缘质量中心
  → 计算质量中心偏离画面中心的距离
  
score = max(0, 1 - distance × 1.5)
```

- 主体偏离中心越远，评分越低
- 允许一定程度的偏离（三分法构图）

#### 综合评分

```
total = brightness × 0.25 + sharpness × 0.30 + color × 0.25 + composition × 0.20
```

## 3. 人脸识别

**文件**: `src/faces/` | **框架**: MediaPipe Face Detection

### 3.1 MediaPipe Face Detection

Google MediaPipe 的轻量级人脸检测模型。

#### 模型特点

| 参数 | 值 |
|------|-----|
| 模型 | BlazeFace（单发检测器） |
| 模式 | `model_selection=1`（全身模型） |
| 关键点 | 6 个（眼睛、耳朵、鼻子、嘴巴） |
| 置信度阈值 | 0.5 |
| 速度 | ~20ms/张（CPU） |

#### `model_selection` 说明

| 值 | 模型 | 适用场景 |
|----|------|---------|
| 0 | 近距离模型 | 自拍、半身照 |
| **1** | **全身模型** | **旅行照片中的远距离人脸** |

#### 输出格式

```python
{
    "bbox": [x, y, w, h],      # 人脸框（像素坐标）
    "keypoints": [              # 6个关键点
        {"x": 100, "y": 80},   # 左眼
        {"x": 140, "y": 80},   # 右眼
        {"x": 120, "y": 100},  # 鼻尖
        {"x": 100, "y": 110},  # 左嘴角
        {"x": 140, "y": 110},  # 右嘴角
        {"x": 120, "y": 100},  # 左耳
    ],
    "confidence": 0.95          # 置信度
}
```

#### 精度与局限

| 场景 | 精度 | 说明 |
|------|------|------|
| 正面人脸 | ~99% | 检测非常可靠 |
| 侧面人脸 | ~85% | 有一定漏检 |
| 戴墨镜/口罩 | ~60% | 关键点可能不准 |
| 极远距离 | ~40% | 人脸太小可能漏检 |
| 合照（10人+） | ~80% | 重叠人脸可能漏检 |

## 4. 场景分类

**文件**: `src/scene/` | **模型**: CLIP 零样本分类

### 4.1 零样本分类原理

CLIP 的零样本分类不需要针对场景类别进行微调，直接通过文本匹配进行分类。

#### 算法步骤

```
输入: 图片 + 16个场景类别文本
  │
  ├─ 1. 图片通过 ViT 编码 → 512 维图片嵌入
  ├─ 2. 每个场景类别文本通过 Transformer 编码 → 512 维文本嵌入
  ├─ 3. 计算图片嵌入与所有文本嵌入的余弦相似度
  └─ 4. Softmax 归一化 → 每个类别的概率
```

#### 预定义的 16 个场景类别

| 英文 | 中文 | 典型照片 |
|------|------|---------|
| landscape | 风景 | 山川、草原 |
| cityscape | 城市 | 街景、高楼 |
| architecture | 建筑 | 古建筑、地标 |
| food | 美食 | 菜品、餐厅 |
| portrait | 人像 | 单人特写 |
| group photo | 合影 | 多人合照 |
| sunset | 日落 | 黄昏、晚霞 |
| beach | 海滩 | 海边、沙滩 |
| mountain | 山景 | 登山、远眺 |
| street | 街道 | 小巷、马路 |
| night scene | 夜景 | 灯光、星空 |
| indoor | 室内 | 博物馆、商场 |
| nature | 自然 | 森林、花卉 |
| wildlife | 野生动物 | 动物、鸟类 |
| macro | 微距 | 花朵特写、昆虫 |
| aerial view | 航拍 | 无人机视角 |

#### 输出示例

```python
{
    "风景": 0.45,    # 最高概率 → 主要场景
    "自然": 0.25,
    "山景": 0.15,
    "城市": 0.08,
    ...
}
```

## 5. 地点聚类

**文件**: `src/geocluster/` | **数据源**: EXIF GPS + 拍摄时间

### 5.1 GPS 提取

从照片 EXIF 数据中读取 GPS 信息。

#### EXIF GPS 标签

| 标签 | 说明 | 示例值 |
|------|------|--------|
| `GPSLatitude` | 纬度 (度分秒) | (39, 54, 15) |
| `GPSLatitudeRef` | 纬度方向 | N/S |
| `GPSLongitude` | 经度 (度分秒) | (116, 24, 27) |
| `GPSLongitudeRef` | 经度方向 | E/W |
| `DateTimeOriginal` | 拍摄时间 | "2026:05:01 14:30:00" |

#### 度分秒 → 十进制转换

```
十进制 = 度 + 分/60 + 秒/3600

示例: (39, 54, 15) → 39 + 54/60 + 15/3600 = 39.9042°
```

### 5.2 Haversine 距离

计算两个经纬度坐标之间的球面距离。

```
a = sin²(Δlat/2) + cos(lat1)·cos(lat2)·sin²(Δlon/2)
c = 2 · arcsin(√a)
distance = R · c    # R = 6371 km（地球半径）
```

### 5.3 地点聚类

将距离在阈值以内的照片归为同一地点。

```
输入: 带 GPS 的照片列表
  │
  ├─ 1. 提取所有照片的 GPS 坐标
  ├─ 2. 贪心聚类:
  │     对每张未分配的照片:
  │       找到所有距离 < threshold 的照片
  │       归为一组
  └─ 3. 命名: "地点1(39.90, 116.41)"
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `distance_km` | 1.0 km | 在此距离内的照片视为同一地点 |

### 5.4 时间聚类

将拍摄时间在阈值以内的照片归为同一时间段。

```
输入: 带时间的照片列表
  │
  ├─ 1. 按时间排序
  ├─ 2. 顺序扫描:
  │     如果当前照片与组内第一张的时间差 < threshold
  │       加入当前组
  │     否则
  │       新建一组
  └─ 3. 命名: "时段1(05-01 14:30-15:45)"
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `hour_threshold` | 2 小时 | 在此时间内的照片视为同一时段 |

### 5.5 混合聚类

优先使用 GPS 聚类，无 GPS 的照片使用时间聚类。

```
输入: 所有照片
  │
  ├─ 有 GPS? → 地点聚类 → "地点1(39.90, 116.41)"
  │
  └─ 无 GPS? → 时间聚类 → "时段1(05-01 14:30-15:45)"
```

## 6. Pipeline 编排

**文件**: `src/pipeline/orchestrator.py` + `run_pipeline.py`

### 6.1 完整流程

```
输入目录（3000张原始照片）
  │
  ├─ Stage 1: collect_images()
  │   rglob 递归搜索所有 jpg/jpeg/png/heic
  │
  ├─ Stage 2: hybrid_dedup()
  │   dHash 粗筛 → CLIP 精筛 → ~500 张
  │
  ├─ Stage 3: rank_by_aesthetics()
  │   NIMA 评分 → 取 top 100
  │
  └─ Stage 4: copy_results()
      复制精选到输出目录
```

### 6.2 入口命令

```bash
# 完整流程
python run_pipeline.py --input ./photos --output ./selected --top-k 100

# 仅去重（不需要 torch）
python run_pipeline.py --input ./photos --output ./selected --dedup-only

# 仅评分排序
python run_pipeline.py --input ./photos --output ./selected --rank-only
```

### 6.3 支持的图片格式

| 格式 | 支持 | 说明 |
|------|------|------|
| `.jpg`/`.jpeg` | ✅ | 完全支持 |
| `.png` | ✅ | 完全支持 |
| `.heic` | ⚠️ | 需要额外安装 `pillow-heif` |

## 7. 参数调优指南

### 7.1 常用调参场景

| 场景 | 调整参数 | 建议值 |
|------|---------|--------|
| 照片数量少（<500） | `clip_threshold` | 0.95（更严格，保留更多） |
| 连拍很多 | `dhash_threshold` | 3（更敏感，删更多） |
| 想要更多精选 | `top_k` | 150-200 |
| 想要更少精选 | `top_k` | 50-80 |
| 无 GPU | 自动 | 自动使用 CPU，速度慢但可用 |
| 注重清晰度 | 美学评分权重 | 可修改 heuristic.py 中的权重 |

### 7.2 速度参考（CPU，i7-12700）

| 阶段 | 3000 张 | 1000 张 | 100 张 |
|------|---------|---------|--------|
| dHash 去重 | ~30s | ~10s | ~1s |
| CLIP 去重 | ~5min | ~1.5min | ~10s |
| NIMA 评分 | ~7.5min | ~2.5min | ~15s |
| 启发式评分 | ~1.5min | ~30s | ~3s |
| 人脸检测 | ~1min | ~20s | ~2s |
